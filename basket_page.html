Determining Which Items Are Often Sold Together

The Apriori Algorithm iterates through all possible combinations of items, counting how many times each set appears, comparing it to the sales as a whole, and taking into account how many times such a combination is likely to occur by chance.
Here is a more detailed description of the metrics used in the results.
Create a unique product id by combining Category id with Subcategory id
Each transaction only included 1 type of item. Ideally, we would like to see which items are purchased at the same time for product placement purposes and promotional sales. Since this dataset did not allow for that, we will group by Customer ID. Even if the items were not purchased at the same time, it will be useful to know what items people are likely to purchase later.
The data is changed to a binary format to be used in the algorithm and represents either a purchase or no purchase.
